qemu

1. QEMU terms

2. QEMU调用

Usage: qemu-system-x86_64 [options] [disk_image]
一般使用[options]：
-h：帮助
-version：查看版本
-M machine：选择虚拟机,-machine -h可查看支持的虚拟机
-cpu model：选择cpu型号，-cpu help
-smp [cpus=]n[,cores=cores][,threads=threads][,dies=dies][,sockets=sockets][,maxcpus=maxcpus]：指定cpu个数、核数、线程数
-m [size=]megs[,slots=n,maxmem=size]：指定内存大小，默认为128MiB，可以通过后缀M或G指定megs单位
-boot：启动选项，软盘（a）、硬盘（c）、光驱（D）、网卡（n），默认是从硬盘启动
# 指定启动的linux内核：使用以下选项时，能直接指定启动的内核而不必按照在磁盘镜像中，对测试内核节省了时间
-kernel bzImage:使用bzImage作为启动的内核镜像
-append cmdline：启动内核时传递的参数
-initrd file：使用file文件作为初始化ram文件系统
-initrd "file1 arg=foo,file2"
-dtb file：设备树文件
# 指定驱动器
-drive option[,option[,option[,...]]]：定义一个新的驱动器，有效选项如下：
 file=file：定义镜像文件
 if=interface：该选项定义驱动被连接的类型，ide, scsi, sd, mtd, floppy, pflash, virtio.
 bus=bus,unit=unit：这些选项定义连接的驱动器定义总线数字和单位ID。
 index=index：此选项定义驱动器连接在一个给定的接口类型列表中可用的连接器使用索引。（可能为盘符）
 media=media：cdrom还是disk
 cyls=c,heads=h,secs=s[,trans=t]：选项同‘-hdachs’
 snapshot=snapshot：值为"on" or "off"，允许被给的驱动器使能snapshot
 cache=cache：值为"none", "writeback", or "writethrough" ，控制主机缓存用于访问数据块
 aio=aio：值为"threads", or "native" ，选择线程基于磁盘IO还是本地linux IO
 format=format：指定磁盘格式，建议使用raw
 serial=serial：给设备指定序列号（可能是串口，也可能和串口无关）
 addr=addr：指定PCI地址（仅仅：if=virtio）
    默认情况下，连续写入缓存被用于所有的块设备，这意味着主机页缓存将被用于读和写数据，但是仅仅当数据被报知被写入存储子系统，写通知将被发送到客户机。
    Instead of ‘-cdrom’ you can use:qemu -drive file=file,index=2,media=cdrom
    Instead of ‘-hda’, ‘-hdb’, ‘-hdc’, ‘-hdd’, you can use:
	qemu -drive file=file,index=0,media=disk
	qemu -drive file=file,index=1,media=disk
	qemu -drive file=file,index=2,media=disk
	qemu -drive file=file,index=3,media=disk
# 显示
-nographic：只使用命令行

-fda file：
-fdb file：使用文件作为软盘，可以使用主机的“/dev/fd0”
-hda file：硬盘选项
-hdb file：硬盘选项
-hdc file：硬盘选项
-hdd file：硬盘选项
-cdrom file：光驱选项



-snapshot：快照功能
-no-fd-bootchk：禁止启动检测软盘
-numar opts：模拟一个多节点的非统一内存访问，如果cpu内存都被忽略，资源平分
-nographic：通常，QEMU使用SDL显示VGA输出，使用这个选项，使qemu成为简单的命令行应用程序
-k language：键盘布局，默认是en-us
-audio-help：声音子系统帮助显示，驱动列表，可调参数等
-soundhw card1,card2,... or -soundhw all：使能音频并选择声卡
-localtime：设置真实时钟为本地时钟，默认是UTC时间
-full-screen：全屏
-pidfile file：存储qemu进程PID到文件，用于脚本启动
-daemonize：初始化后QEMU进程长期驻留，qemu将不从标准IO分离，除非他准备连接到他的设备中的起其中一个。这个选项用于外部程序启动qemu
而不用去初始化静态条件。
-option-rom file：加载一个可选ROM文件内容，用于加载像EtherBoot这样的东西。
-mtdblock file：

术语

CPU：中央处理器单元，是个大的概念集合，包含了实际处理指令的物理核core、内存控制器、PCIe控制器等。

core：实际执行一条控制流的硬件执行单元，其含有独立的一整套寄存器。

die：指处理器生产过程中从晶圆上切割下来的一个个小方块，电路逻辑被光刻机刻在上面，一个CPU可以包含一个及以上的die，一个die可以含有一个及以上的core。

package：指包含一个或多个die的塑料/陶瓷外壳和镀金的触点，以配合你的主板。主板上的每个CPU插槽（socket），只能接受一个package。

hexo使用

这里介绍采用hexo搭建完blog后的常用使用方法，如何搭建参考官方手册。

1 hexo常用命令

# 安装hexo命令
npm install hexo -g

# 升级hexo
$ npm update hexo -g

# 建站
$ hexo init <folder>
$ cd <folder>
$ npm install

# 新建文章
$ hexo new "post_name"

# 生成
$ hexo generate

# 启动预览服务，在本地游览blog
$ hexo server

# 部署
$ hexo deploy

# 清除缓存
hexo clean

2 主题next的配置

1 启用分类(categories)

【1】：新建对应的页面，执行命令后会在source\categories下生产index.md文件

$ hexo new page "categories"

【2】：编辑index.md文件，原始index.md文件内容如下：

title: categories
date: 2021-04-18 15:34:05

新增type字段，其值为“categories”：

title: categories
date: 2021-04-18 15:34:05
type: categories

【3】：在站点配置文件_config.yml增加菜单

menu:
  home: /
  archives: /archives

【4】：在blog的头部增加categories字段，并设定其分类，如

title: hexo
date: 2021-04-18 13:23:33
categories: 博客搭建

如果是多分类则如下：

title: hexo
date: 2021-04-18 13:23:33
categories: 
 - 博客搭建
 - example

2 启用标签(tags)

【1】：新建对应的页面，执行命令后会在source\categories下生产index.md文件

$ hexo new page "tags"

【2】：编辑index.md文件，原始index.md文件内容如下：

title: tags
date: 2021-04-18 15:34:05

新增type字段，其值为“tags”：

title: tags
date: 2021-04-18 15:34:05
type: tags

【3】：在主题配置文件_config.yml增加菜单，如本blog采用next主题，菜单路径为themes\next

menu:
  home: /
  archives: /archives
  tags: /tags

【4】：在blog的头部增加tags字段，并设定其标签，如

title: hexo
date: 2021-04-18 13:23:33
tags: hexo

如果是多标签则如下：

title: hexo
date: 2021-04-18 13:23:33
tags: 
 - hexo
 - example

3 文章目录

在低版本的next主题中，会遇到文章目录，即toc，仅显示一级目录且无法跳转的问题。因此建议采用较新的版本，或者在网上搜索能正常显示文章目录且跳转的blog所采用的next版本。本blog当前采用v8.3.0版本。

【1】:在主题配置文件\themes\next_config.yml中禁用自动分配目录等级

toc:
  enable: true
  # Automatically add list number to toc.
  number: false # 改变为false
  # If true, all words will placed on next lines if header width longer then sidebar width.
  wrap: false
  # If true, all level of TOC in a post will be displayed, rather than the activated part of it.
  expand_all: false
  # Maximum heading depth of generated toc.
  max_depth: 6

【2】：文件目录总是显示：

sidebar:
  # Sidebar Position.
  position: left
  
  # Sidebar Display (only for Muse | Mist), available values:
  #  - post    expand on posts automatically. Default.
  #  - always  expand for all pages automatically.
  #  - hide    expand only when click on the sidebar toggle icon.
  #  - remove  totally remove sidebar including sidebar toggle.
  display: always

注：点击文章两边的留白，文章目录会自动收缩。

4 文章版权声明

在主题配置文件\themes\next_config.yml中按如下设置

creative_commons:
  license: by-nc-sa
  sidebar: true # 在边栏增加版权图片
  post: true # 在文章末尾增加版权信息
  language:

5 首页头像

在主题配置文件\themes\next_config.yml中找到avatar字段，取消其注释，并在\themes\next\source\images\目录下放入头像的图片，如avatar_self.jpg，并改变url字段的值

# Sidebar Avatar
avatar:
  # Replace the default image and set the url here.
  url: /images/avatar_self.jpg
  # If true, the avatar will be dispalyed in circle.圆形头像
  rounded: true
  # If true, the avatar will be rotated with the cursor.光标悬浮头像会旋转
  rotated: true

6 取消文章底部带”#”号标签

在主题配置文件\themes\next_config.yml中找到tag_icon字段，并按如下所示修改

# Use icon instead of the symbol # to indicate the tag at the bottom of the post
tag_icon: true

7 统计游览人数

在主题配置文件\themes\next_config.yml中找到busuanzi_count字段，并按如下所示修改

busuanzi_count:
  enable: true
  total_visitors: true
  total_visitors_icon: fa fa-user
  total_views: true
  total_views_icon: fa fa-eye
  post_views: false
  post_views_icon: far fa-eye

8 搜索功能

【1】：安装插件

$ npm install hexo-generator-searchdb --save

【2】：修改站点配置文件

# Search Config
search:
  path: search.json #之前就是这里出了问题导致图标错误，错误的教程是 search.xml
  field: post
  format: html
  limit: 1000

【3】：修改主题配置文件

# Local Search
# Dependencies: https://github.com/theme-next/hexo-generator-searchdb
local_search:
  enable: true # 改为true
